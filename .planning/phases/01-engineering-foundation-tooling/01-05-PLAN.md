```yaml
# .github/dependabot.yml
version: 2

updates:
  # Root npm packages
  - package-ecosystem: "npm"
    directory: "/"
    schedule:
      interval: "weekly"
      day: "monday"
    open-pull-requests-limit: 10
    rebase-strategy: "auto"
    # Auto-merge is intentionally NOT configured — D-19 compliance posture requires
    # human review on every dependency bump. Re-evaluate post-milestone-close.

  # functions/ workspace (forward-declared for Phase 7 — Cloud Functions)
  # Dependabot silently skips ecosystems whose directory does not yet exist.
  - package-ecosystem: "npm"
    directory: "/functions"
    schedule:
      interval: "weekly"
      day: "monday"
    open-pull-requests-limit: 5
    rebase-strategy: "auto"

  # GitHub Actions — keeps SHA pins in .github/workflows/ci.yml fresh
  - package-ecosystem: "github-actions"
    directory: "/"
    schedule:
      interval: "weekly"
      day: "monday"
    open-pull-requests-limit: 5
    rebase-strategy: "auto"
```

Load-bearing constraints:

- `version: 2` — Dependabot config schema version (mandatory)
- Three `- package-ecosystem:` entries — VALIDATION.md TOOL-10 row Wave 4 expects count == 3
- Weekly cadence (D-19) — daily was rejected in CONTEXT.md as too noisy
- `open-pull-requests-limit: 10` for root npm (large dep surface), `5` for functions and github-actions (smaller surfaces)
- `rebase-strategy: "auto"` — Dependabot keeps each open PR current with `main`
- NO `automerge:` key — D-19 explicitly disallows auto-merge for compliance posture
- The `/functions` directory does NOT exist in Phase 1; Dependabot silently skips ecosystems whose directory is absent — verified by GitHub Dependabot docs
