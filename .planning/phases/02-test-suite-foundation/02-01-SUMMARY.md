## Accomplishments

- **Vitest setup file wired and load-bearing.** `tests/setup.js` runs in every worker before every test: seeds fake timers at `2026-01-01T00:00:00.000Z`, replaces `crypto.randomUUID` with a monotonic counter producing valid UUIDv4-shaped strings (`00000000-0000-4000-8000-{hex(N)}`), pins `Math.random` to `0.5`, stubs `matchMedia` / `ResizeObserver` / `IntersectionObserver`, and `vi.mock`s `chart.js` with `makeChartStub`. All Phase 2 downstream tests inherit these defaults.
- **Reusable mock factories landed.** `tests/mocks/firebase.js` exports `makeFirestoreMock` (D-11) — to be reused by Plan 02-05's `cloud-sync` + `migration` tests and Plan 02-04's `unread` test. `tests/mocks/chartjs.js` exports `GlobalChartStub` (constructor capturing `config` on `__lastConfig`) + `makeChartStub` (vi.mock factory) so radar / dashboard snapshot tests can assert chart wiring without rendering.
- **Auth-password fixture pipeline established.** `tests/fixtures/_generators/hash-passwords.js` (Node CLI, `node tests/fixtures/_generators/hash-passwords.js`) regenerates SHA-256 hex from three known plaintexts; `tests/fixtures/auth-passwords.js` is the committed output. Wave 4 (Plan 02-05) auth state-machine tests assert against these fixtures by importing them — no per-test hashing required.
- **Preflight 3 (crypto parity) PASSED.** `tests/crypto-parity.test.js` confirms happy-dom 20.9.0's `crypto.subtle.digest("SHA-256", ...)` produces byte-identical hex to Node's `createHash("sha256").update(...).digest("hex")` for all three known plaintexts. This unblocks Wave 4's decision to use the same primitive in `src/util/hash.js` without a separate test environment.
- **ESM bridge landed (D-04 supersedes Phase 1 D-14).** `index.html` line 26 now reads `<script type="module" src="./app.js?v=50"></script>` (was: `<script defer src="app.js?v=50">`). `defer` removed (modules are deferred by default per HTML spec). Leading `./` retained because Vite 8 + GH-Pages both interpret bare `app.js` differently from `./app.js` on subpath routes.
- **GH-Pages Pre-flight 1 confirmed live.** Worktree branch `worktree-agent-aad976d0f4cf7d725` pushed to `origin/phase-2-bridge-test`; user flipped GH-Pages source and verified at `https://baselayers.bedeveloped.com`. DevTools Network tab returned `content-type: application/javascript; charset=utf-8` (with `accept-ranges: bytes`, `content-encoding: gzip`, `cache-control: max-age=600`, `etag: W/"69fa5a0f-2d885"`, served via `varnish` x-cache HIT — all GH-Pages defaults). Console clean, page rendered. T-2-01 (Tampering / Denial of Service via wrong MIME type) mitigated end-to-end.
- **Zero new npm packages introduced.** `git diff 9bf53fd2..HEAD -- package.json package-lock.json` is empty. Plan-level constraint (and D-15) honoured.

2. **Rule 3 (anti-pattern correction):** new `tests/crypto-parity.test.js` and `tests/fixtures/_generators/hash-passwords.js` failed `tsc --noEmit` with `Cannot find name 'node:crypto'`-style errors because they import Node-only APIs. Fix: added `tests/fixtures/_generators` and `tests/crypto-parity.test.js` to `tsconfig.json` `exclude` (Node CLI tooling + Node-using preflight; mirrors the existing exclude entries for `vite.config.js` and `eslint.config.js`). Avoids adding `@types/node` which would violate D-15 (zero new npm packages). Logged for cleanup ledger.
3. **Rule 1 (lint compliance):** `tests/setup.test.js` line 21 needed to assert `Math.random() === 0.5`, which the project's `no-restricted-syntax` rule forbids globally. Fix: added `// eslint-disable-next-line no-restricted-syntax -- intentional: this test verifies the Math.random spy in tests/setup.js` (matches the pattern `app.js:33` already uses for the same rule). The exemption is narrow (one line, one assertion) and self-documenting.
