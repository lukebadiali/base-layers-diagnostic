## Accomplishments

- **Closed the load-bearing pre-flight finding from RESEARCH § Critical Pinned Fact 1.1:** Admin SDK `auth.createUser({password})` bypasses Identity Platform `passwordPolicy` (≥12 chars + HIBP). Without the length floor on `setOrgClientPassphrase`, Wave 2's `inviteClient.ts` body would succeed at user creation, but the invited client would be bricked at first `signInWithEmailAndPassword` with `auth/password-does-not-meet-requirements`. The chokepoint gate now sits at `src/auth/passphrase-policy.js` + the `setOrgClientPassphrase` call site + the modal's submit handler — defence-in-depth via the same helper at three layers.
- **Established the AUTH-12 chokepoint error surface for the invite flow:** 3 new classes (`PassphraseInvalidError`, `CrossOrgError`, `PassphraseNotSetError`) land at the same `src/firebase/auth.js` chokepoint as `SignInError` / `MfaRequiredError` / `MfaCodeInvalidError`. The user-facing message strings are pinned by tests and consumed verbatim by the Wave 2 Invite Client modal — no Firebase error codes will leak to views.
- **Pure-logic test seam for the inviteClient branching logic:** `decideInviteOutcome` is a 5-variant discriminated-union switch over `(passphraseSet, passphraseValid, existingUser, requestedOrgId)` — fully unit-testable without firebase-functions-test or Admin SDK mocks. The 6-case boundary table includes the orphan-claims adoption edge case from RESEARCH § 8.
- **Pattern A callable skeleton ships with the explicit `claims-admin-sa` service account pin** (vs setClaims.ts which uses the default runtime SA) — establishes the SA contract Wave 2 will need without scope-creeping into the body work. The skeleton's `unimplemented` HttpsError throw ensures any accidental Wave 1 invocation fails loudly.
- **4 new audit-event enum entries appended** (`auth.client.invite`, `.resend`, `.rejected.cross-org`, `.rejected.passphrase-invalid`) — Wave 2's callable body writes through these via the existing `writeAuditEvent` substrate. Existing 79 enum entries preserved in order (snapshot tests still green).
