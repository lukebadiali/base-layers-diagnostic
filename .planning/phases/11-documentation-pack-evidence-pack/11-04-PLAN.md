```markdown
# Credential Compromise — Incident Response Runbook

**Trigger:** Slack alert from `authAnomalyAlert` Rule 1 (auth-fail burst) OR Rule 3 (role escalation), OR direct report from internal user, OR Sentry error spike on `auth.*` events.
**Owner (default):** Hugh (BeDeveloped on-call). Escalation: notify Luke + George within 1 hour.
**Severity:** Critical if admin account; High if internal non-admin; Medium if anonymous-bound.
**Last reviewed:** 2026-05-10 (Phase 11 Wave 4 — skeleton; operator fills on first use)

## Pre-conditions

- [ ] Operator has `firebase` CLI authenticated to bedeveloped-base-layers project
- [ ] Operator has Firebase Console access at https://console.firebase.google.com/project/bedeveloped-base-layers/authentication/users
- [ ] Operator has Cloud Logging access at https://console.cloud.google.com/logs

## Decision tree

1. **Confirm compromise.** Review Slack alert payload + audit-log entries from `authAnomalyAlert` (look for: anomalous IP, off-hours role escalation, MFA disenrol followed by sensitive action).
2. **Disable affected user.** Via Firebase Console → Users → disable, OR via `firebase auth:export users.json && firebase auth:import users.json --hash-algo=...` for batch.
3. **Revoke all sessions.** `firebase auth:revoke-tokens <uid>`.
4. **Audit-log review (24h prior).** `gcloud logging read 'jsonPayload.event=~"role|mfa|gdpr"' --freshness=24h --limit=200`.
5. **Re-enable + re-issue credentials.** Email-link recovery per Phase 6 D-07 / `phase6-mfa-recovery-drill.md` Tier 1; if email itself compromised, Tier 2 Admin SDK un-enrol.
6. **Post-incident.** Capture timeline; fill RCA template (see `docs/IR_RUNBOOK.md ## RCA template`); update this runbook with lessons learned.

## Failure-Mode Triage

| Failure | Cause | Recovery |
|---------|-------|----------|
| `auth:revoke-tokens` returns "user not found" | UID typo or already deleted | Verify with `firebase auth:export` |
| Slack webhook delivers but operator missed alert | Slack channel notification routing | Check `runbooks/phase-9-monitors-bootstrap.md` Step 4; escalate to phone/SMS pager v2 |
```
